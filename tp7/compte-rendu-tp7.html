Exercice 1  : Memory 

  1. Ouvrez un navigateur Web et allez à l'URL suivante: http://www.w3.org/standards/
    Quel est le nom de la machine à laquelle vous accédez ?
    Quel est le nom de domaine auquel vous accédez ?

    • Nom de domaine : est w3.org
    • Nom de machine : est www.w3.org plus précisement le "www".

  2. Si l'on exécute host w3.org
    À quoi correspond la première ligne de la réponse ?
    À quoi correspond la deuxième ligne de la réponse ?
    Et la troisième ?

    Cette commande interroge le DNS. Les trois lignes correspondent généralement à :

    1. Adresse IPv4 : L'identifiant numérique classique donc ici 104.18.23.19.
    2. Adresse IPv6 : L'identifiant nouvelle génération donc ici 2606:4700::6812:1713.
    3. Serveur Mail : Le serveur qui gère les courriels pour ce domaine donc ici "5 puck.w3.org".
    david@Davs:~$ host w3.org
w3.org has address 104.18.23.19
w3.org has address 104.18.22.19
w3.org has IPv6 address 2606:4700::6812:1713
w3.org has IPv6 address 2606:4700::6812:1613
w3.org mail is handled by 5 puck.w3.org.
w3.org mail is handled by 5 pan.w3.org.

  3. Dans le navigateur Web, dans la barre contenant l'URL, remplacez le nom de la machine par son adresse IP pour vérifier que cela fonctionne toujours.
    Si ça ne marche pas, que se passe-t-il ? Essayez avec google.fr.
    À votre avis, pourquoi utilise-t-on des noms pour les machines plutôt que des adresses IP ?

    Si l'on remplace le nom par l'IP, cela peut échouer (Erreur 404). Cela arrive car un même serveur (une seule IP) peut héberger des centaines de sites différents. Sans le nom de domaine, le serveur ne sait pas quel site afficher. Donc on utilise des noms pour la mémorisation humaine et la flexibilité technique (on peut changer de serveur/IP sans changer l'adresse URL).

  4. Exécutez la commande traceroute sur une de ces adresses IP.
    Qu'affiche cette commande ?
    Comment s'appelle le réseau sur lequel est connectée l'université (ou votre domicile) ?
    Quels sont les autres réseaux qui sont traversés ?
    Si vous êtes sur Mac, il se peut que votre traceroute se coince, cela peut aider d'ajouter l'option -I (par exemple traceroute -I 8.8.8.8).

    Cette commande affiche tous les routeurs intermédiaires. À l'université, on passe généralement par le réseau RENATER, puis par des opérateurs de transit (comme Orange ou Cogent) avant d'atteindre la destination.

  6. On cherche maintenant à voir l'ancienne version du cours, que donnait Rémi Forax, à l'adresse https://forax.github.io/course/progweb/. Lancer la commande host pour récupérer l'adresse IP correspondant à cette machine. Qu'observe-t-on ? Que se passe-t-il si on essaye d'accéder à cette page dans le navigateur en remplaçant le nom de la machine par l'adresse IP trouvée précédemment ? Et si on remplaçait forax.github.io par vincentjuge1987.github.io au début de l'URL ? Expliquer pourquoi.

    On observe que vincentjuge1987.github.io et forax.github.io partagent la même adresse IP.

    En remplaçant le nom par l'IP, le serveur est incapable de distinguer quel site servir. Si on change le nom dans l'URL mais garde le chemin, le serveur cherchera la page de Rémi Forax sur le compte de Vincent Juge, ce qui provoquera une erreur.
