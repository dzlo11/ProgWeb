Exercice 1 : Uniform Resource Locator

  1. Ouvrez un navigateur Web et allez à l'URL suivante: http://www.w3.org/standards/
    Quel est le nom de la machine à laquelle vous accédez ?
    Quel est le nom de domaine auquel vous accédez ?

    • Nom de domaine : w3.org.
    • Nom de machine : www (souvent utilisé pour désigner le serveur Web par défaut).

  2. Si l'on exécute host w3.org
    À quoi correspond la première ligne de la réponse ?
    À quoi correspond la deuxième ligne de la réponse ?
    Et la troisième ?

    La commande interroge le service DNS (Domain Name System) pour traduire le nom en adresses exploitables par le réseau :

    1. Première ligne : correspond à l'adresse IPv4 (format 32 bits, ex: 104.18.23.19). C'est l'adresse standard utilisée pour router les paquets sur Internet.
    2. Deuxième ligne : correspond généralement à une seconde adresse IPv4 (pour la redondance) ou à l'adresse IPv6 (format 128 bits).
    3. Troisième ligne : indique le serveur de messagerie (MX Record). C'est la machine responsable de la réception des emails pour le domaine w3.org.
    david@Davs:~$ host w3.org
w3.org has address 104.18.23.19
w3.org has address 104.18.22.19
w3.org has IPv6 address 2606:4700::6812:1713
w3.org has IPv6 address 2606:4700::6812:1613
w3.org mail is handled by 5 puck.w3.org.
w3.org mail is handled by 5 pan.w3.org.

  3. Dans le navigateur Web, dans la barre contenant l'URL, remplacez le nom de la machine par son adresse IP pour vérifier que cela fonctionne toujours.
    Si ça ne marche pas, que se passe-t-il ? Essayez avec google.fr.
    À votre avis, pourquoi utilise-t-on des noms pour les machines plutôt que des adresses IP ?

    En essayant d'accéder directement via l'IP 104.18.23.19, on obtient souvent une erreur (type 403 ou 404) de la part de Cloudflare. Cela se produit car le serveur héberge plusieurs sites sur la même IP ; sans le nom d'hôte, il ne sait pas quel site afficher.

    On utilise des noms car :

    • Mémorisation : Il est plus simple de retenir "google.fr" qu'une suite de chiffres.
    • Flexibilité : L'administrateur peut changer de serveur (et donc d'IP) sans que l'utilisateur n'ait à changer l'adresse qu'il tape.
    • Mutualisation : Une seule adresse IP peut servir des milliers de sites différents.

Exercice 2 : Service HTTP

  1. Quel est le port TCP correspondant au protocole HTTP ? Quel est le header de la réponse ?

    Le port standard est le 80. L'en-tête (header) est l'ensemble des lignes de métadonnées renvoyées avant le contenu HTML.

    ngalulakabongo@pcader102-10:~$ nc -C monge.univ-mlv.fr 80
GET /ens/ HTTP/1.0

HTTP/1.1 200 OK
Date: Fri, 03 Apr 2026 08:30:55 GMT
Server: Apache
Content-Length: 2998
Content-Type: text/html; charset=UTF-8

<html>
<head><title>Enseignements</title></head>
<body>...</body>
</html>

  2. À quoi correspondent les 3 valeurs sur la première ligne de l'en-tête ?

    1. HTTP/1.1 : La version du protocole utilisé.
    2. 200 : Le code d'état numérique (Status Code) indiquant que la requête a réussi.
    3. OK : La phrase de raison (Reason Phrase) décrivant le code de statut.

  3. Quel est le nom du serveur Web ?

    Le serveur utilisé est Apache (indiqué par le champ Server: Apache).

  4. À quoi correspond la valeur de Content-Type ?

    Elle indique le type MIME du document. Ici, text/html signifie que le contenu est une page HTML encodée en UTF-8.

  5. À quoi correspond la valeur de Content-Length ?

    Elle indique la taille du corps de la réponse (la page HTML elle-même) en octets. Ici, le fichier pèse 2998 octets.

  6. Faites la même analyse mais avec la requête GET /ens/toto.txt HTTP/1.0

    Le serveur répond par un code 404 Not Found. Cela signifie que la connexion au serveur a réussi, mais que la ressource spécifique demandée n'existe pas sur le serveur.

    ngalulakabongo@pcader102-10:~$ nc -C monge.univ-mlv.fr 80
GET /ens/toto.txt HTTP/1.0

HTTP/1.1 404 Not Found
Date: Fri, 03 Apr 2026 08:31:35 GMT
Server: Apache
Connection: close
Content-Type: text/html

<HTML><TITLE>404 Not Found</TITLE></HTML>

  7. Enfin, faites la même analyse avec la requête GET /index.html HTTP/1.0. À quoi correspond la valeur de Location ?

    Le serveur répond par un code 302 Found (redirection temporaire). La valeur de Location (ici http://ligm.univ-eiffel.fr/index.html) indique l'URL vers laquelle le navigateur doit automatiquement se diriger pour trouver la page.

    ngalulakabongo@pcader102-10:~$ nc -C monge.univ-mlv.fr 80
GET /index.html HTTP/1.0

HTTP/1.1 302 Found
Date: Fri, 03 Apr 2026 08:32:17 GMT
Server: Apache
Location: http://ligm.univ-eiffel.fr/index.html
Content-Length: 291
Connection: close
Content-Type: text/html; charset=iso-8859-1

  8. Que se passe-t-il lorsque je lance la commande wget monge.univ-mlv.fr/index.html ? Expliquer la signification des lignes.

    La commande wget télécharge automatiquement le fichier. Les étapes affichées sont :

    1. Résolution de l'adresse : Le client cherche l'IP associée au nom.
    2. Connexion : Établissement de la liaison sur le port 80.
    3. Requête HTTP envoyée : Le client demande le fichier.
    4. Réponse du serveur : Réception du code (302 puis 200 après redirection).
    5. Sauvegarde : Le contenu est écrit dans un fichier local nommé index.html.
